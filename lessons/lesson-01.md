## 📚 핵심 개념

### 1. C#과 .NET의 관계
- **C#**: 프로그래밍 언어 → 요리 레시피에 비유
- **.NET**: C# 코드를 실행시켜주는 시스템 → 요리사 + 주방 + 도구에 비유
- **C# 코드만 있고 .NET이 없으면** 실행도 컴파일도 불가능

### 2. .NET이 해주는 일
1. C# 코드를 컴퓨터가 이해하는 기계어로 번역 (컴파일)
2. 수천 개의 편리한 기본 기능 제공 (파일 읽기, 네트워크 통신, 날짜 계산 등)
3. 메모리 관리, 에러 처리 같은 귀찮은 작업 대신 처리
4. Windows/Linux/macOS 같은 운영체제와 프로그램 연결

### 3. .NET의 역사
| 시기 | 이름 | 특징 |
|---|---|---|
| 2002~2019 | .NET Framework | Windows 전용 |
| 2016~2019 | .NET Core | 크로스플랫폼 (Windows/Mac/Linux) |
| 2020~현재 | .NET 5, 6, 7, 8... | 통합 시대, 그냥 ".NET" |

- 2020년 **.NET 5**부터 Framework와 Core가 하나로 통합
- **버전 4는 건너뜀** (옛날 .NET Framework 4.x와 헷갈려서)

### 4. .NET 8의 특징
- **2023년 11월 출시**
- **LTS (Long-Term Support, 장기 지원) 버전**
- 지원 기간: **2026년 11월까지 (3년)**
- **짝수 버전 = LTS = 안정적으로 오래 사용 가능**
- 실무 회사들은 주로 LTS 버전 사용

### 5. Windows / .NET / WPF / C# 계층 구조
```
┌─────────────────────────────────┐
│         Windows (운영체제)         │  ← 프로그램이 사는 땅
├─────────────────────────────────┤
│    .NET 8 (C#을 실행하는 시스템)    │  ← 주방 + 요리사 + 도구
├─────────────────────────────────┤
│     WPF (화면을 그리는 도구)       │  ← 창, 버튼, 차트를 만드는 붓
├─────────────────────────────────┤
│   C# (프로그래밍 언어)             │  ← 레시피를 쓰는 언어
└─────────────────────────────────┘
```

### 6. WPF란?
- **W**indows **P**resentation **F**oundation의 줄임말
- Windows에서 화면(UI)을 그리는 기반 기술
- C#으로 Windows 데스크톱 앱의 화면을 만들 때 사용하는 **도구 세트**
- **C# 언어 자체가 아니라**, C#으로 사용할 수 있는 "도구 모음"

### 7. 왜 회사 개발팀은 C# WPF를 사용할까?
1. **Windows 데스크톱 앱이 필요한 상황** (상황실 모니터링 프로그램)
2. **Microsoft 진영의 찰떡궁합**: Windows, C#, .NET, Visual Studio 모두 Microsoft 제품
3. **복잡한 실시간 UI에 강함**: 레이더 화면, 실시간 그래프, 지도, CCTV 영상 등을 한 화면에 동시 표시
4. **24시간 안정적 운영**: 상황실 환경에 적합

## 🔑 기억할 용어

| 용어 | 뜻 |
|---|---|
| **C#** | Microsoft가 만든 프로그래밍 언어 |
| **.NET** | C# 코드를 실행시켜주는 시스템 |
| **.NET 8** | 2023년 11월 출시, LTS, 2026년 11월까지 지원 |
| **LTS** | Long-Term Support, 장기 지원 버전 (3년) |
| **.NET Framework** | 옛날 Windows 전용 닷넷 (2002~2019) |
| **.NET Core** | 크로스플랫폼 닷넷 (2016~2019) |
| **WPF** | Windows Presentation Foundation, Windows UI 도구 |
| **데스크톱 앱** | PC에 설치해서 쓰는 프로그램 (웹/모바일 앱의 반대) |
| **컴파일** | 사람이 쓴 코드를 컴퓨터가 이해하는 언어로 번역하는 것 |

## 🌱 나중에 배울 WPF 핵심 기술 (미리 단어만)
**7개월 차**에 자세히 배울 예정:
- **데이터 바인딩**: 데이터가 바뀌면 화면이 자동으로 따라 바뀌는 기술
- **의존성 속성**: 수많은 값이 실시간으로 변해도 효율적으로 처리하는 기술
- **Dispatcher**: 여러 작업이 동시에 돌아가도 화면이 안 멈추게 하는 기술
