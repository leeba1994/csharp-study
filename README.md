# csharp-study

풀스택 개발자로의 여정을 기록하는 저장소입니다.

## 👤 나는 누구인가

CCTV 유지보수 업무를 하며 개발자의 꿈을 키우는 사람입니다.
현장 경험을 무기로 실무에 바로 투입 가능한 **주니어 중급 이상 풀스택 개발자**가 되는 것이 목표입니다.

## 🎯 최종 목표

**약 30개월 안에 번호판 OCR 통합 관제 시스템 완성하기**

- **1차 목표 (18개월)**: C# WPF 기반 실시간 번호판 OCR 관제 프로그램 완성
- **2차 목표 (30개월)**: 백엔드 서버 + 웹 대시보드 + 모바일 앱까지 풀스택 확장

## 🗺️ 학습 로드맵

| Phase | 기간 | 주요 내용 |
|---|---|---|
| Phase 1 | 1~5개월 | C# 언어 완전 정복 (콘솔) |
| Phase 2 | 6~9개월 | WPF + GitHub 협업 + MVVM |
| Phase 3 | 10~12개월 | 디자인 패턴 + OpenCvSharp |

| Phase 5 | 19~21개월 | 자바 속성 + Spring Boot 기초 |
| Phase 6 | 22~24개월 | Spring Boot 심화 + JPA + REST API |
| Phase 7 | 25~27개월 | React 기초 + 실전 |
| Phase 8 | 28~30개월 | React Native + 풀스택 완성 |

## 📝 커밋 메시지 규칙

모든 커밋은 아래 4가지 접두사 중 하나로 시작한다.

| 접두사 | 언제 쓰나 | 예시 |
|---|---|---|
| `feat:` | 새 기능 추가 | `feat: 번호판 검증 함수 추가` |
| `fix:` | 버그 수정 | `fix: 소수점 계산 오류 수정` |
| `docs:` | 문서 작성/수정 | `docs: README 작성` |
| `study:` | 공부 기록 | `study: Lesson 3 디버깅 예제 작성` |
